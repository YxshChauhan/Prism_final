## Overview

AirLink features a **Zapya-inspired** modern interface built with Material 3 design principles. The UI is designed to be intuitive, fast, and visually appealing while maintaining functionality across different screen sizes and platforms.

### Design Principles
- **Simplicity First** - Clear, uncluttered interface
- **Visual Feedback** - Immediate response to user actions
- **Consistency** - Uniform design language throughout
- **Accessibility** - Support for different abilities and preferences
- **Performance** - Smooth 60fps animations

#### Platform Compliance

**Android Permission Model:**
- Follow Android 13+ granular permissions
- Request `READ_EXTERNAL_STORAGE` for file access
- Use `MANAGE_EXTERNAL_STORAGE` for full file system
- Implement `REQUEST_INSTALL_PACKAGES` for APK installation
- Follow Android's permission rationale guidelines

**OAuth UX Best Practices:**
- Use system browser for OAuth flows
- Implement PKCE for mobile OAuth
- Store tokens securely using Android Keystore
- Implement proper token refresh mechanisms
- Follow OAuth 2.1 security recommendations

**Security Indicators Placement:**
- **Top-right corner**: Encryption status badge
- **File list items**: Signature verification icons
- **Transfer progress**: Security status indicators
- **Settings page**: Security configuration options
- **App bar**: Overall security status indicator

**Required Permission Text:**
- Use clear, non-technical language
- Explain why each permission is needed
- Provide examples of what the app will do
- Include links to privacy policy and terms
- Allow users to grant permissions incrementally

**Success/Error States:**
- **Success**: Green checkmark with confirmation message
- **Error**: Red warning with specific error details
- **Loading**: Spinner with progress indication
- **Retry**: Clear retry button with error explanation
- **Fallback**: Alternative action when primary fails
